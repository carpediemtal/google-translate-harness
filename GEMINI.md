# Keyboard Shortcuts for Google Translate (KS4GT) - Refactored

## Project Overview
This project is a browser extension (Chrome and Firefox) that adds customizable keyboard shortcuts to the Google Translate website. It has been modernized from a Grunt-based setup to a modern Vite + TypeScript + Tailwind CSS architecture.

### Core Technologies
- **TypeScript:** Primary language for all extension logic (background, content, and options).
- **Vite:** Modern bundler and development server, using `@crxjs/vite-plugin` for extension-specific handling.
- **Tailwind CSS:** Utility-first CSS framework used for the Options page.
- **PostCSS:** Used for CSS processing (Autoprefixer and Tailwind).

### Architecture
- `src/`: Core source code.
  - `src/background/`: Background service worker (`index.ts`).
  - `src/content/`: Content scripts (`index.ts`) injected into Google Translate.
  - `src/options/`: Options page logic and UI (`options.html`, `index.ts`, `style.css`).
- `dev/`: Legacy static assets and JSON settings used by the extension.
- `dist/`: Build output directory generated by Vite.
- `manifest.json`: Root extension manifest.
- `vite.config.ts`: Vite configuration for the extension.
- `tailwind.config.js` & `postcss.config.js`: Styling configurations.

## Building and Running

### Development
To start the Vite development server with Hot Module Replacement (HMR) for the options page and automatic rebuilding of scripts:
```bash
npm run dev
```

### Production Build
To create a production-ready build in the `dist/` directory:
```bash
npm run build
```

## Development Conventions
- **TypeScript:** All new logic should be written in TypeScript within the `src/` directory.
- **Styling:** Use Tailwind CSS for the Options page. Content script styling currently relies on existing CSS files in the `dev/` directory, which are injected dynamically.
- **Manifest:** Modify the root `manifest.json` for any permission or entry point changes.
- **Assets:** Static assets like images and default settings are located in `dev/` and are included in the build via the manifest.
